# 01 PRD / Requirements

## 1. 想定ユーザー
- メイン: 女子高生・女子大生（SNS慣れ／友達同士で回す）
- サブ: 20代前半

## 2. ユースケース
- 友達と「どれ出た？」で盛り上がる
- 恋愛/距離感/メンタルのヒントが欲しい
- 勉強・やる気の出し方を知りたい
- 結果をシェアしたい（画像/URL）

## 3. 体験要件（Awwwards系）
- 3D/アニメは“主役”ではなく背景としてUIを邪魔しない
- マウスホバーで微細に反応（parallax / magnetic / tilt）
- スクロールで段階的に見せる（progressive reveal）
- ボタン/カードに「押せた感」「跳ね返り」

## 4. 機能要件（MVP）
### 診断フロー
- [F-01] スタート（目的・所要時間・注意書き）
- [F-02] 48問（例: 12問×4ページ）
- [F-03] 途中保存（localStorage）
- [F-04] 回答品質チェック（注意問題・矛盾検知）
- [F-05] 結果生成（スコア＋派生）
- [F-06] 結果ページ（Screen A〜D）
- [F-07] シェアカード生成（PNG）

### 結果内容
- [F-08] アーキタイプ（12種）+ 連続スコア（裏付け）
- [F-09] Trait 6軸（0-100）
- [F-10] アタッチメント4象限
- [F-11] SDT 3欲求（満たされ度）
- [F-12] “今日の一手”クエスト（3つ）

## 5. 非機能要件
- モバイル最適化（3D軽量／停止）
- WebGL不可でフォールバック
- prefers-reduced-motion 対応
- 断定禁止・医療ではない注意書き
